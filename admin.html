Instruções para configurar no n8n:

  1. Adicione um módulo HTTP Request ao seu workflow n8n
  2. Defina o método como GET
  3. Cole a URL do endpoint acima no campo URL
  4. Alternativa (este site): GET /api/n8n-products — ?flat_images=1 lista URLs para WhatsApp (1 foto por produto por padrão). Todas as fotos por SKU: &all_photos=1. Autenticação: header x-n8n-products-secret (mesmo valor de N8N_PRODUCTS_SECRET na Vercel).
  5. Adicione o header Content-Type: application/json
  6. Use a resposta JSON no módulo AI Agent ou OpenAI do n8n como contexto
  7. Você também pode usar o webhook configurado nas Configurações para receber dados automaticamente
  8. O webhook envia summary_line (uma linha com nome, preço, categoria, tags, trecho da descrição) e o objeto product completo
  9. Mesma IA do chat do site: HTTP Request POST na URL abaixo (catálogo completo no servidor — não use OpenAI separado no n8n).
    —
    Chaves na Vercel (não .env local): OPENAI_API_KEY, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY. Não é necessário header de autorização neste POST. Body → JSON, campo em modo Expression (com =). Após o webhook do admin, use $json inteiro: ={{ JSON.stringify({ user_message: 'Gere legenda promocional', product: $json, include_catalog: true, site_base_url: 'https://confortacolchoes.vercel.app' }) }} Se o produto estiver em $json.product: troque por product: $json.product. Legenda curta: ={{ JSON.stringify({ n8n_product_blurb: true, product: $json.product || $json, instruction: 'Legenda Instagram', include_catalog: true, site_base_url: 'https://confortacolchoes.vercel.app' }) }} Resposta: reply. Webhook das Configurações só avisa produto salvo; o chat usa esta API.
